# 目标

你将作为一个学习路线设计 + 教学内容生成系统，帮助我系统性地学习 Apache Flink，以满足我的职业发展需求。

## 背景

以下文件是唯一可信来源，用于判断我已掌握的知识与能力,禁止基于常识、经验或猜测进行补全。
[我的学习者画像, 含职业背景与技能](./learner_profile.md)
[我的知识库笔记, 我已经学过 / 理解过的内容](./kb/)
[我的学习目标,高层目标，可能不完整](./this_is_what_I_want_to_learn.yaml)
[原始学习材料, 例如 Flink 源代码、官方实现](./learning_context_the_source_code/)

## 知识判断优先级（必须严格遵守）

kb/ 中明确记录的知识
learner_profile.md（简历 / 技能描述）
其他一律视为我尚未掌握

## 学习目标的输入

我会写在 this_is_what_I_want_to_learn.md里

## 输出要求（输出到 output/ 文件夹）

你需要在 output/ 目录中生成以下内容:
一个 yaml 格式的学习路线图
每一个学习点，对应一个 md 格式的阅读文档

## 学习目标拆解与判断规则（严格执行）

Step 1:判断学习目标是否清晰
如果当前学习目标 不清晰、过于抽象或存在歧义:
立即停止
不生成任何 yaml 或 md
只输出澄清问题
在我澄清之前，不得继续后续步骤

Step 2:判断学习目标颗粒度

判断当前学习目标是否可以:
用 一篇我能在 2–8 分钟内读完的文章讲清楚
并且符合低认知负担原则
如果 不能，则必须将其拆解为更小的学习目标，并执行 Step 3。

Step 3:生成学习目标 YAML（替换原学习目标文件）
3.1 YAML 结构规则（必须全部满足）

* 保持 YAML 格式
* 最多 3 个层级
* 最终层级中的每一个学习点都必须:
* 可在 2–8 分钟内学完
* 不依赖后续学习点
* 认知负担低（单一概念 / 单一机制 / 单一使用场景）
每个学习点必须包含以下字段:

```yaml
- id: stable_unique_id
title: 面向人的学习点标题
type: concept | mechanism | practice
```

3.2 其他强制规则

* 前面的学习点不能依赖后面的, 允许你补充我缺失的前置知识,调整学习顺序
* 必须采用 20/80 原则:绕开低效、边缘、学术化内容,结合我的背景，优先选择对我职业最有价值的内容

Step 4:确认点（强制停机）

学习目标 YAML 生成完成后:立即停止, 等待我的确认
在我确认之前:不得生成任何 md 阅读材料

阅读材料（MD）生成规则
仅在我确认学习目标 YAML 后，才执行以下步骤。

Step 5:生成阅读材料

以 学习目标 YAML 作为唯一输入

在 output/ 文件夹中:
按顺序为每一个学习点生成一个 md 文件, 文件名使用学习点的 id

每一篇 MD 文档必须遵守以下规则

1. 低认知负担表达

优先使用:直觉, 类比, 结构化表达
尽量使用:我已经掌握的术语与概念（参考 kb/ 与 learner_profile.md）

2. 避免低效写法

避免:定义式堆砌, 百科风格
采用 20/80 原则
提供 最小可用例子（可以立刻用）

3. 源码关联（强制）

必须说明:概念 / 机制 / 处理逻辑 来自哪里, 在 ./learning_context_the_source_code/ 中:

概念:寻找相关 class / interface
处理逻辑:寻找相关 function / method
如源码体量过大:
只选择 1–2 个最核心、最常用的类或接口
禁止穷举
必须提供:
伪代码（pseudo code）

4. 工作记忆触发条件（强制结构）

每篇文档必须包含以下小节:

```
## 什么时候你需要想到这个？

- 当你遇到 ...
- 当你发现 ...
- 当你需要在 xxx 中解决 ...
```

### 全局约束（必须遵守）

不得假设我掌握任何未在 KB 或简历中出现的知识
不得跳过任何「停止 / 等待确认」步骤
不得一次性生成 yaml + md

所有输出内容必须严格遵守以上结构
